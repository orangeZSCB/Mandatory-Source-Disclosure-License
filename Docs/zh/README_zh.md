## 这是什么

MSDL（强制源代码披露许可证）是一个比 AGPL 更激进的 Copyleft 许可证。

它的核心主张只有一条：**修改即须开源，无论你有没有分发。**

大多数开源许可证（包括 GPL、AGPL）的开源义务都由"分发"触发——只要你不把软件交给别人，你在内部怎么改都没有义务开源。MSDL 取消了这个前提——改了就得开源。

## 与主流许可证的对比

| 行为 | MIT | GPL v3 | AGPL v3 | **MSDL** |
|------|-----|--------|---------|----------|
| 直接使用（不改动） | ✅ 无限制 | ✅ 保留声明 | ✅ 保留声明 | ✅ 保留声明 |
| 修改后内部使用，不分发 | ✅ 无限制 | ✅ 无义务 | ✅ 无义务 | ⚠️ **须开源** |
| 修改后作为 SaaS 提供 | ✅ 无限制 | ✅ 无义务 | ⚠️ 须开源 | ⚠️ **须开源** |
| 修改后分发 | ✅ 无限制 | ⚠️ 须开源 | ⚠️ 须开源 | ⚠️ **须开源** |
| 配置文件/密钥 | ✅ 无限制 | ✅ 无义务 | ✅ 无义务 | ✅ **豁免** |

## 你可以做什么

- ✅ 使用本软件（原样，不加改动）
- ✅ 分发本软件（须保留许可证及版权声明）
- ✅ 修改配置文件、填写 API 密钥等部署操作
- ✅ 将本软件作为不加改动的依赖项引用
- ✅ 基于本软件进行学习、研究

## 你不可以做什么

- ❌ 修改源代码后，以任何理由拒绝公开源代码
- ❌ 在衍生版本上施加比本许可证更严格的限制
- ❌ 删除或遮蔽原版权声明和本许可证文本
- ❌ 将本软件的修改版以闭源形式分发或部署

## 常见问题

**Q：我只是改了 `.env` 里的数据库密码，也要开源吗？**

不需要。配置文件、环境变量、运行时生成的数据文件均在豁免范围内（第二条）。只要你没动源代码逻辑，就不触发开源义务。

**Q：我在公司内部部署，只有自己用，也要开源吗？**

是的。MSDL 的开源义务不依赖"分发"，内部使用同样适用。这是 MSDL 与 GPL/AGPL 最根本的区别。

**Q：我改了一点点，开源方式有什么要求？**

至少满足以下一项：发布到公开代码托管平台（GitHub/GitLab/Gitee 等）、随副本附带完整源代码、或提供书面承诺按需提供源代码（有效期三年）。

**Q：我违规了，还有补救机会吗？**

有。首次违规可在收到通知后 30 日内纠正，权利自动恢复。二次及以上违规须版权持有人书面同意方可恢复（第七条）。

**Q：这个许可证算"开源"吗？**

取决于定义。它不符合 OSI 的开源定义（OSI 要求不限制使用场景），但源代码是完全公开的，精神上属于强 Copyleft 阵营。如果你在意 OSI 认证，AGPL v3 是最接近的替代选择。

## 如何使用本许可证

在你的项目根目录放置 `LICENSE` 文件（使用本许可证文本），并将文件头部的占位符替换：

```
Copyright (c) 2025 你的名字或组织
```

在项目 README 中注明：

```
本项目基于 MSDL v1.0 许可证发布。详见 LICENSE 文件。
```

## 设计理念

代码是公共财产的延伸。你可以在它的基础上建造，但你建造的部分，也应属于所有人。

MSDL 不是为了阻止人们使用软件，而是为了确保改进能够回流到公共领域——无论改进发生在哪里。
